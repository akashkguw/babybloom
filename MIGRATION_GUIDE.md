# BabyBloom Migration Guide: Monolith → Modular

## Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Start dev server
npm run dev

# 3. Type-check
npm run type-check

# 4. Build for production
npm run build

# 5. Preview production build
npm run preview
```

## What Changed

### Before (v1)
- Single 2,537-line `index.html` with all logic
- `React.createElement` via CDN — no JSX, no build step
- Manual `sw.js` cache version bumps (`babybloom-v26`)
- Regex-based test extraction with `eval()` in Node
- String-matching CI checks

### After (v2)
- 58 modular TypeScript/JSX files across `src/`
- Vite build with `@vitejs/plugin-react` and `vite-plugin-pwa`
- Auto-generated service worker with content-hash cache busting
- Vitest for unit/integration testing
- TypeScript compilation as CI gate

## Data Continuity

IndexedDB schema is **identical** to v1. Existing user data migrates seamlessly — no data conversion needed. The same `dg()`, `ds()`, `dga()`, `dcl()` functions are used, just typed.

## Key Files for Common Changes

| Task | File(s) |
|------|---------|
| Add a new log category | `src/tabs/LogTab.tsx` + add form fields |
| New milestone age group | `src/lib/constants/milestones.ts` |
| New guide section | `src/lib/constants/guides.ts` + `src/tabs/GuideTab.tsx` |
| Change theme colors | `src/lib/constants/colors.ts` |
| Add voice command | `src/features/voice/parseVoice.ts` |
| New shared component | `src/components/shared/` + update `index.ts` |
| Change vaccine schedule | `src/lib/constants/vaccines.ts` |
| Add i18n locale | `src/lib/constants/i18n.ts` |

## PWA / Service Worker

Service worker is now auto-generated by `vite-plugin-pwa`. No more manual `sw.js` edits. Cache busting uses content hashes. Configuration is in `vite.config.ts`.

## Testing

```bash
npm run test        # Run all tests

npm run lint        # ESLint
```

## Multi-Country Support

The `src/lib/constants/i18n.ts` file provides locale support for:
- `en-US` (default): Imperial units, 911, US date format
- `en-IN`: Metric units, 112, DD/MM/YYYY
- `hi-IN`: Hindi labels, metric, Indian emergency numbers

Wire into components by importing `getLocale()` and using locale-specific formatting.
